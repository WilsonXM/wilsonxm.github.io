【Definition】一棵AVL树是其每个节点的左子树和右子树的高度最多差一的BST（空树的高度定义为-1）

【Definition】Balance Factor：BF(node) = Height(left) - Height(right)

所以在一棵AVL树中，BF（node）为1，-1，0

[1.2.1.x] 红黑树——AVL树变种

【Definition】满足下面五条性质的 BST 称为红黑树：

  1. 每个节点非红即黑；
  2. 根节点必须着黑色；
  3. 每个叶子节点为黑色 (NIL/NULL)；
  4. 红色节点的儿子必须是黑色的；
  5. 从任意节点开始到叶子节点上的所有路径上包含相同数量的黑色节点；

📣 注意：

  1. 性质c中的“叶子节点”指的是哨兵节点(外部节点)；

【Definition】 The black-height of any node x——bh(x), is the number of black nodes on any simple path from x (x not included) down to a leaf. bh(Tree) = bh(root).

【Definition】 A red-black tree with N internal nodes has height at most 2ln(N +1).

  2. 除根外，所有非树叶节点的儿子数在和 M 之间；

【Definition】满足下列性质的二叉树称为左倾堆（树）（leftist heaps）：

堆序性质：与二叉堆相同，可分为最小堆/最大堆；

【Definition】 The null path length, Npl(X), of any node X is the length of the shortest path from X to a node without two children. Define Npl(NULL) = –1.

【NOTE】

结构性质：对堆中的每个节点X，其左右儿子满足 ；

【Theorem】 A leftist tree withnodes on the right path must have at least nodes.

【Note】 The leftist tree ofnodes has a right path containing at mostnodes.

【Merge】——O(logN)

--recursive version

  1. Merge( H1->Right, H2 )->递归入口
  2. Attach( H2, H1->Right )
  3. Swap(H1->Right, H1->Left ) if necessary
  4. 递归出口->Merge时出现一棵空树

--iteration version

  1. Sort the right paths without changing their left children；
  2. Swap children if necessary；

注意：自底向上Swap更新，如果自顶向下Swap会改变right path上的节点使得下一步没法进行下去；

[1.4.2.x] Skew Heaps——leftist heap的简单版本

【Definition】满足下列性质的二叉树称为斜堆（skew heaps）：

堆序性质：与二叉堆相同，可分为最小堆/最大堆；

结构性质：每次Merge操作都交换左右子树，除非当前节点是右路径上的最大节点；

【Advantage】：Skew heaps have the advantage that no extra space is required to maintain path lengths and no tests are required to determine when to swap children.

【Note】：It is an open problem to determine precisely the expected right path length of both leftist and skew heaps.

[1.5.1.x] Binomial Queue

【目的】

在左偏树和斜堆中插入一个元素的时间复杂度为O(logN)，但是构建一个N个元素二叉堆的时间复杂度为O(N)，均摊到每个元素就是O(1)，因此二项队列的目的是达到常数时间的插入操作并保持较好的合并操作复杂度；

【Definition】满足下列性质树称为二项队列（Binomial Queue）：

  1. 满足堆的顺序性质，最小元素/最大元素；
  2. 结构性质上，二项队列不是一棵堆序树，而是堆序树的森林，此时森林中的每棵树被称为二项树

方法：将size表示为二进制数

【FindMin】

Solution1：最小元素存在于森林中的树根上，所以遍历树根，时间复杂度为O(logN);

Solution2：维护一个最小元素，在进行merge或者delete操作时更新；

【Merge】

基本思想：合并高度相同的树，类似二进制加法；>> 二项队列中的树需保持高度排序

Q1：为何选择”左儿子右兄弟“而不是普通的二叉树？

A1：因为二项队列的目的在于简化查询操作，不在意访问队列中的任意一个元素，所以只需要左儿子右兄弟；

Q2：为何选择从左到右从大到小的高度安排？

A2：从merge操作考虑，两个高度相同的树相连，从小到大意味着需要遍历其中一个树的所有子树从而找到最大的那个，并将新的最大子树塞进去，而从大到小只需要直接将根节点较大的子树放在开头就行；

从delete角度考虑，删除一个元素后，不管是高到低还是低到高都需要将被删除最小root的树中的所有字数都遍历一遍放入队列中，因此没有区别；

[1.6.1.x] Backtracking——Theory

解决一个问题的可靠想法是穷举所有可能情况，逐一验证，排除错误情况，找到所有正确的答案，但是这样的方法不仅效率极低，而且对可能情况很多的问题耗时过大，因此我们需要更聪明的方法。

【基本思想】回溯使我们能够中途结束对某些已知不可能情况的彻底的显示检查（pruning：在一步内删除一大组可能性）；

【基本方法】

  1. 设已有部分答案，其中，；
  2. 将加入到答案中，检查是否可能(satisfy the constraints)：
    1. 若可能，则继续加入下一个;
    2. 若不可能，则从答案中删除，并尝试下一个；如果所有的都不满足条件，则回溯到答案；

【Template代码】

/* 参数 i 表示当前答案的规模 */
bool Backtracking ( int i )
{   Found = false;
    if ( i > N )
        return true; /* solved with (x1, …, xN) */
    for ( each xi in Si ) { 
        /* check if satisfies the restriction R */
        OK = Check((x1, …, xi) , R ); /* pruning */
        if ( OK ) {
            Count xi in;
            Found = Backtracking( i+1 );
            if ( !Found )
                Undo( i ); /* recover to (x1, …, xi-1) */
        }
        if ( Found ) break; 
    }
    return Found;
}

【 α - β pruning】MinMax剪枝中的常见策略

以α-剪枝为例，在44的位置我们需要一个最大值，而在40的level对方会选择最小值，此时如果我们探测到40的左儿子是40则不需要继续探测右儿子，因为如果右儿子小于40，则44的右儿子会比40小，那么最大值就是左儿子44，如果40的右儿子大于40，则44的右儿子还是40，则根节点选最大值依然是44，故可以直接剪枝；β-剪枝同理相反；

【The Turnpike Reconstruction Problem】

【解题思路】

根据给出的距离个数，可以通过计算出需要的站点个数N，然后关注最大距离可以确定出首尾两个点，其次再寻找次大距离进行迭代；

若无剪枝，则该算法的时间复杂度为，首尾元素唯一，中间距离的元素都有两种可能；

若有剪枝，则

【代码实现】

bool Reconstruct ( DistType X[ ], DistSet D, int N, int left, int right )
{ /* X[1]...X[left-1] and X[right+1]...X[N] are solved */
    bool Found = false;
    if ( Is_Empty( D ) )
        return true; /* solved */
    D_max = Find_Max( D );
    /*//---- option 1：X[right] = D_max ----//*/
    /* check if |D_max-X[i]| is in D for all X[i]s that have been solved */
    OK = Check( D_max, N, left, right ); /* pruning */
    if ( OK ) { /* add X[right] and update D */
        X[right] = D_max;
        for ( i=1; i<left; i++ )  Delete( |X[right]-X[i]|, D);      // 从D删去新加入的点产生的新distance
        for ( i=right+1; i<=N; i++ )  Delete( |X[right]-X[i]|, D);  // 从D删去新加入的点产生的新distance
        Found = Reconstruct ( X, D, N, left, right-1 );             // 递归下一层
        if ( !Found ) { /* if does not work, undo */
            for ( i=1; i<left; i++ )  Insert( |X[right]-X[i]|, D);      // 消除新加入点对D的影响
            for ( i=right+1; i<=N; i++ )  Insert( |X[right]-X[i]|, D);  // 消除新加入点对D的影响
        }
    }
    /* finish checking option 1 */
    if ( !Found ) { /* if option 1 does not work */
        /*//---- option 2: X[left] = X[N]-D_max ----//*/
        OK = Check( X[N]-D_max, N, left, right );
        if ( OK ) {
            X[left] = X[N] – D_max;
            for ( i=1; i<left; i++ )  Delete( |X[left]-X[i]|, D);       // 从D删去新加入的点产生的新distance
            for ( i=right+1; i<=N; i++ )  Delete( |X[left]-X[i]|, D);   // 从D删去新加入的点产生的新distance
            Found = Reconstruct (X, D, N, left+1, right );
            if ( !Found ) {
                for ( i=1; i<left; i++ ) Insert( |X[left]-X[i]|, D);        // 消除新加入点对D的影响
                for ( i=right+1; i<=N; i++ ) Insert( |X[left]-X[i]|, D);    // 消除新加入点对D的影响
            }
        }
        /* finish checking option 2 */
    } /* finish checking all the options */

    return Found;
}

【Closest Points Problem】

Given N points in a plane.  Find the closest pair of points.  (If two points have the same position, then that pair is the closest with distance 0.)

采用分治算法：

类似于最大子序列的分类方法；考虑左边、右边、横跨左右三种情况，递归的解决左右两边，并在合并时解决横跨两边；

此时，，时间复杂度为；

因此下面考虑对该算法进行类似剪枝的操作，不必次遍历；

第一次剪枝：

  1. 先递归解决左右两边规模为的点，找出此时最小距离**δ**；
  2. 遍历距离中线距离小于 δ 所有点；

第二次剪枝：

  1. 考虑Y-轴方向的剪枝，需要对Y-轴方向的点排序，由于同时需要进行X-轴方向的剪枝，具体实现上可以维护两个队列；

  2. 最坏情况不存在N-1个点都塞在这个方框里，否则在分治一边的点时 δ 就会变小；最差是塞七个点（中线上会有重叠，且同一边可能会有点被重复考虑）；
